| **典型场景** | 企业文档、FAQ、产品手册、政策法规      | 对话历史、用户偏好、情景记忆        |
| **对标系统** | RAGFlow Corpus、Dify RAG Engine        | LangGraph `Store`、ADK `MemoryBank` |
| **存储表**   | `knowledge`                            | `memories` + `facts`                |

### Chat

Chat 页的对话内容需要能够做到「绝对的所见即所得」，即即时地显示最终正确内容的正确效果，而不是在用户输入信息后，流式响应过程中，对话内容先杂乱或错误显示，每次都需要人工刷新页面才能得到真正的最终内容和显示效果，而且最终显示的内容还缺失了响应过程的展示，比如思考的过程、工具调用的过程、子 Agent 之间的协作与委派过程等等。。。
注意以上功能特性是基于 AGUI、A2UI 两个层面的协议，以及 ADK 的工程实践支撑完成的，你需要最大可能的标准化、与最佳实践的方式来应用 AGUI、A2UI、ADK 这三者，从而达到上述效果。
